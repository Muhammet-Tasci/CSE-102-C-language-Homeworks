# Assignment 6: Vault Codebreaker Game

## Overview
This project is a dynamic number-guessing game inspired by classic codebreaker scenarios, developed in C for the CSE 102 Computer Programming course. The program challenges players to deduce a secret numeric code generated by the system. It features two distinct operational modes (Admin and Player), provides real-time character-based feedback (`C`, `M`, `W`) on guesses, and calculates a final score that ranks the player.

## Key Features
* **Dual Operation Modes:** * **Admin Mode (`A`):** Allows an administrator to dynamically configure game rules (e.g., code length, digit ranges, duplicate allowances, max attempts, and scoring metrics) and saves them persistently to `vault_config.txt`.
  * **Player Mode (`P`):** Loads the pre-configured rules, generates a random secret code (saved to `vault_code.txt`), and engages the user in a guessing loop until the code is broken or attempts run out.
* **Real-Time Feedback & Scoring:** Evaluates each digit of the player's guess to provide specific feedback: Correct digit/Correct place (`C`), Correct digit/Wrong place (`M`), and Wrong digit (`W`). The score is dynamically adjusted based on the admin's point configuration.
* **Comprehensive Logging:** Utilizes the `<time.h>` library to timestamp games, writing every guess, feedback string, and the final rank (e.g., Code Master, Cipher Hunter) to a detailed `game_log.txt` file.

## Technical Implementation
This assignment heavily emphasizes advanced C programming concepts:
* **Dynamic Memory Allocation:** Strictly avoids global or static arrays for storing the code and guesses. Instead, it utilizes `malloc()` within custom functions (`generate_code()` and `get_guess()`) to allocate exact memory sizes based on the admin's chosen code length, subsequently freeing (`free()`) the memory in `main()` to prevent leaks.
* **Pointer Arithmetic & Returns:** Functions are designed to return integer pointers (`int*`) to pass dynamically created arrays safely back to the main execution loop.
* **Complex Validation Logic:** Implements robust error handling and `do-while` loops to ensure admin configurations are logical (e.g., preventing impossible constraints like generating a 5-digit unique code using only numbers 1-3).
